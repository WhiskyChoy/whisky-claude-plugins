# Sync: Commit + Pull + Push

Full bidirectional sync with the Overleaf remote in three steps.

## Step 1: Commit

```bash
git add -A
git commit -m "<auto-generated meaningful message>"
```

Commit message format: summarize what changed (e.g., "Fix overfull hbox in sections 3-5, adjust float placement").

If there are no changes to commit, skip to Step 2.

## Step 2: Pull with Rebase

```bash
git pull --rebase origin master
```

If conflicts:
- Show conflicting files to user
- Offer to help resolve (LaTeX-aware merge — understand that `\section`, `\begin{figure}`, etc. are structural boundaries)
- **Do not auto-resolve** — academic content conflicts require human judgment

## Step 3: Push

Confirm with user before pushing: "Ready to push N commits to Overleaf. Proceed?"

```bash
git push origin master
```

## Edge Cases

- **Concurrent Overleaf edits:** Collaborator edited while user worked locally. Rebase may conflict — show conflicts, help resolve, never auto-resolve content.
- **Large binary files:** Push may be slow with many images. Warn about push size.
- **Overleaf git bridge downtime:** Push/pull fails → detect, suggest retry later.
- **Auth token expired:** Pull/push fails with auth error → guide user to regenerate token at Overleaf → Account Settings → Git Integration.
